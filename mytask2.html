Git(Windows.ver)

git教程

软件
msysgit-是Windows版的Git
创建版本库
首先，选择一个合适的地方，创建一个空目录：

$ mkdir learngit
$ cd learngit
$ pwd
/Users/xxx/learngit
*pwd命令用于显示当前目录。这个仓库位于/Users/xxx/learngit。
*使用Windows系统，为了避免遇到各种莫名其妙的问题，请确保目录名（包括父目录）不包含中文。


第二步，通过git init命令把这个目录变成Git可以管理的仓库（不可随意手动修改）：

$ git init
Initialized empty Git repository in /Users/michael/xxx/.git/
*如果你没有看到.git目录，那是因为这个目录默认是隐藏的，用ls -ah命令就可以看见。


添加文件
分为add和commit两个步骤

$ git add file1.txt
$ git add file2.txt file3.txt
$ git commit -m "add 3 files."。
*文件必须放在Git仓库目录下（子目录也行），放到其他地方Git再厉害也找不到这个文件。

客户端

【添加文件到仓库】
在test项目文件下，新建一个xx.txt文件，提交到版本库的方法如下2种： 1. 先提到变更列表中，再commit到配置库中，选择新增文件，右键SVN菜单执行“Add“操作提交到”变更列表中”，然后右键SVN菜单执行”SVN Commit”提交到版本库中。 2. 不提交到变更列表中，而是直接commit配置库中，选择该文件，右键svn菜单执行”SVN Commit”操作。

【还原】
右击想要回退的文件或者文件夹，在TortoiseSVN弹出菜单中选择”Update to reversion…” 然后会弹出一个窗口，如下：比如说我们要回退到第10个版本只需要在Revision中填写相应的版本号，然后点击ok即可。

【更新】
更新本地代码与SVN服务器上最新的版本一致，只要在需要更新的文件夹上点击右键或者在文件下空白处点击右键，选择”SVN Update” (获取指定版本中的内容，点击右键执行SVN菜单中的“Update to reversion“)，就可以了。

【改名】
修改文件名，选中需要重命名的文件或文件夹，然后右键“TortoiseSVNRename“，在弹出的对话框中输入新名称，点击”ok”按钮，并将修改文件名后的文件或文件夹通过 “SVN Commit”提交到SVN服务器上。
*会产生两个文件(源文件和更名后的文件)，要同时commit，否则会报错。


总结

SVN是集中式的版本控制系统，而Git是分布式版本控制系统。 集中式版本控制系统最大的毛病就是必须联网才能工作，而且每一步都要commit，要装载的软件比较多，个人感觉用户体验不太好。 和集中式版本控制系统相比，分布式版本控制系统的安全性要高很多， 不用联网，只需安装相应软件然后编写命令就可以传信息，每次修改完的信息会有特殊的标识，用户体验较好
